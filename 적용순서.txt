모스트애드 상품별 진행관리 v15 적용 순서

1. 패치 안의 app, supabase 폴더를 기존 mostad-platform 폴더에 덮어씁니다.
2. Supabase SQL Editor에서 supabase/migrations/005_product_progress_calendar.sql 전체를 실행합니다.
3. 로컬 확인: pnpm dev
4. 최고관리자 화면에서 /admin/work-create 로 업무를 생성하고 직원을 배정합니다.
5. 직원은 /staff 에서 날짜별 링크 또는 순위를 등록합니다.
6. 고객은 /my-marketing 에서 진행률과 날짜별 기록을 확인합니다.
7. 관리자는 /admin/work-calendar 에서 월별 작업 기록을 확인합니다.
8. 배포:
   git add .
   git commit -m "상품별 진행관리 및 작업 캘린더 추가"
   git push

상품 규칙
- 브랜딩 블로그: 10회 220,000P / 20회 440,000P / 30회 660,000P, 포스팅 링크 등록 수로 진행률 계산
- 플레이스 상위노출: 직원이 상위노출 달성일에 보장 시작 체크, 이후 순위 등록일 기준 25일 진행
- 플레이스 관리형: 업무 생성 다음 날부터 25일 관리기간 자동 설정, 직원이 날짜별 순위 등록
- 블로그 상위노출: 24시간 또는 25일 옵션, 날짜별 순위 등록 시 진행 일수 반영
